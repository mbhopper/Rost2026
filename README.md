# Digital Pass API — Цифровой пропуск сотрудника

Сервис для генерации временных цифровых пропусков сотрудников (QR-код) с функциями учёта рабочего времени и обратной связью.  
Разработан для **Ростелеком** в рамках внутреннего MVP.

## 🚀 Основные возможности

- **Регистрация и аутентификация** сотрудников (JWT, bcrypt).
- **Генерация QR-пропуска** (действителен 15 минут).
- **Фиксация статусов** входа/выхода сотрудника.
- **Учёт рабочего времени** (часы работы).
- **Обратная связь** (возможность отправки сообщений администратору).
- **Docker-контейнеризация** для простого развертывания.
- **Защита от брутфорса** (ограничение попыток входа по IP).
- **CORS и Trusted Host** для безопасного взаимодействия с фронтендом.

🧩 Реализованные фишки проекта
1. 🐳 Docker
Полная контейнеризация приложения и базы данных.
Для запуска достаточно иметь установленный Docker и выполнить docker-compose up. Все сервисы поднимаются автоматически.

2. 🚪 Статусы входа и выхода сотрудников
Реализована фиксация каждого входа и выхода сотрудника с временной меткой.
Статус может быть:

check_in — начало рабочего дня

check_out — завершение рабочего дня

Эндпоинты:

POST /api/attendance/check-in

POST /api/attendance/check-out

3. ⏱️ Часы работы
Сотрудник может получить суммарное отработанное время за день или период.
Эндпоинт:

GET /api/attendance/hours?date=2025-03-22

Возвращает количество отработанных часов и минут.

4. 💬 Обратная связь
Пользователи могут отправлять сообщения администратору через защищённый эндпоинт.
Сообщения сохраняются в базе данных с указанием времени и автора.

Эндпоинт:

POST /api/feedback


Формат запроса:
json
{
  "message": "Текст сообщения",
  "subject": "Тема (опционально)"
}

5. 📱 QR-код пропуска
Генерация временного пропуска в виде JSON-строки, которая может быть преобразована в QR-код на фронтенде.
Пропуск содержит уникальный ID, данные сотрудника и время жизни (15 минут).

Эндпоинт:

GET /api/pass/generate

json
{
  "qr_code_data": "{\"pass_id\":\"...\", \"user_id\":\"...\", ...}",
  "message": "Пропуск действителен 15 минут",
  "user_info": {...}
}

6. 🛡️ Защита от брутфорса
Ограничение на количество неудачных попыток входа (5 за 15 минут) по паре IP + username.

🔌 Интеграция с фронтендом
Фронтенд-часть проекта расположена в репозитории:
https://github.com/mbhopper/Rost2026

Для корректной работы:

Убедитесь, что в .env переменная ALLOWED_ORIGINS содержит URL вашего фронтенда (например, http://localhost:3000).

Фронтенд должен отправлять JWT-токен в заголовке Authorization: Bearer <token> для защищённых эндпоинтов.

Все API-эндпоинты имеют префикс /api.

Пример запроса на фронтенде:

javascript

fetch('http://localhost:8000/api/pass/generate', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
